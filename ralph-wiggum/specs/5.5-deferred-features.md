# Spec: Phase 5.5 - Deferred Features

> **Goal:** Implement deep integration features originally planned for Phase 5.
> **Note:** These are lower priority than the UI/UX polish but still part of Phase 5 scope.

## 1. Session Details View
**Requirement:** View full session transcript within the board (Master-Detail style).
- **UI:** Add a "View" button to linked sessions in the Task Details sidebar.
- **Action:** Clicking "View" opens a new pane/modal or replaces the Task Details content with Session content.
- **Implementation:**
  - Create `SessionViewer` component (renders markdown/transcript).
  - Use `useSession(sessionId)` to fetch details.
  - Add navigation breadcrumb: "Task > Session".

## 2. Create Branch from Task
**Requirement:** Create a git branch for a specific task.
- **UI:** "Create Branch" button in Task Actions menu.
- **Action:**
  - Dialog asks for branch name (suggest: `task/{id}-{slug}`).
  - Calls API `POST /api/git/branch`.
- **API:**
  - New endpoint: `src/app/api/git/branch/route.ts`.
  - Logic: `git checkout -b {name}`. (Requires shell execution - SECURITY RISK? Only allow if local).
  - **Constraint:** Since this is a web app, it can only orchestrate. If `opencode` is running, it might have a hook.
  - **Decision:** For now, just generate the *command* and show a "Copy to Clipboard" toast. "git checkout -b ..."
